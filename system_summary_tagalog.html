NCST Enrollment System

Buod ng Sistema at Dokumentasyon - Agosto 13, 2025

Pangkalahatang Impormasyon ng Sistema

Ang NCST Enrollment System ay isang komprehensibong web-based na aplikasyon na dinisenyo upang gawing mas mabilis at mas madali ang proseso ng pagpapatala ng mga estudyante. Ang sistema ay nagbibigay ng serbisyo sa iba't ibang gumagamit tulad ng mga estudyante, registrar, evaluator, at student assistant, na nagbibigay ng isang sentralisadong platform para sa pamamahala ng mga aplikasyon, pagpasa ng mga dokumento, pagpapatala sa kurso, at pagtatasa ng mga bayarin.

Kasalukuyang Bersyon: 1.0.0 (Nasa Development Phase pa)
Huling Update: Pagpapabuti ng interface ng registration forms at proseso ng pagsusuri ng aplikasyon

Istraktura ng Sistema

Istraktura ng Direktoryo

  • /action/ - Backend PHP scripts at interaksyon sa database

Proseso ng Enrollment ng Estudyante

1. Pagpaparehistro ng Estudyante

Ang mga estudyante ay lumilikha ng account at nagsusumite ng personal na impormasyon upang simulan ang proseso ng enrollment.

2. Pagsusumite ng Dokumento

Ang mga estudyante ay nag-a-upload ng mga kinakailangang dokumento batay sa kanilang uri (bago, dating estudyante, transferee, o shifting).

3. Pagtatalaga sa Pila

Ang mga estudyante ay itinatalaga sa isang virtual na pila para sa pagproseso ng mga student assistant at registrar.

4. Pagbeberipika ng Dokumento

Ang mga registrar ay nagbabalidasi ng mga isinumiteng dokumento at minamarkahan ang mga ito bilang kumpleto o humihingi ng karagdagang impormasyon.

5. Pagsusuri ng Aplikasyon

Ang mga registrar ay nagsusuri ng kumpletong aplikasyon at alinman sa nag-aapruba para sa enrollment o tumatanggi kasama ang detalyadong dahilan.

6. Pagpaparehistro sa Kurso

Ang mga estudyante ay pumipili ng mga kurso batay sa mga pangangailangan ng kanilang programa at mga available na iskedyul.

7. Pagtatasa ng Bayarin

Kinakalkula ng sistema ang tuition at iba pang naaangkop na bayarin batay sa mga naka-enroll na kurso at status ng estudyante.

8. Pagproseso ng Bayad

Ang mga estudyante ay pumipili ng opsyon sa pagbabayad (buong bayad o installment) at nagsusumite ng katibayan ng pagbabayad.

9. Kumpirmasyon ng Enrollment

Pagkatapos ng matagumpay na pagbeberipika ng bayad, kinukumpirma ang enrollment, at natatanggap ng mga estudyante ang kanilang mga iskedyul ng klase.

Istraktura ng Database

Ang sistema ay gumagamit ng relational database na may mga sumusunod na pangunahing talahanayan:

students

Nag-iimbak ng personal at akademikong impormasyon ng estudyante

  • id (Primary Key)
  • first_name
  • last_name
  • email
  • contact
  • address
  • type (new, old, transferee, shifting)
  • course
  • year_level
  • created_at
  • updated_at
documents

Sumusubaybay sa mga isinumiteng dokumento ng estudyante

  • id (Primary Key)
  • student_id (Foreign Key)
  • name
  • type
  • file_path
  • status (Submitted, Pending, Missing)
  • date_submitted
  • verified_by
  • verification_date
enrollments

Nagtala ng mga detalye ng enrollment ng estudyante

  • id (Primary Key)
  • student_id (Foreign Key)
  • semester
  • school_year
  • section
  • date_enrolled
  • status
  • approved_by
  • approval_date
courses

Katalogo ng mga available na kurso

  • id (Primary Key)
  • code
  • description
  • units
  • type (Lec, Lab)
  • prerequisites
class_schedules

Mga iskedyul at section ng kurso

  • id (Primary Key)
  • course_id (Foreign Key)
  • section
  • days
  • start_time
  • end_time
  • room
  • instructor
  • max_students
fees

Impormasyon tungkol sa tuition at iba pang bayarin

  • id (Primary Key)
  • enrollment_id (Foreign Key)
  • type
  • amount
  • description
  • payment_status
