System Overview

The NCST Enrollment System is a comprehensive web-based application designed to streamline the student enrollment process. The system caters to different user roles including students, registrars, evaluators, and student assistants, providing a centralized platform for managing applications, document submissions, course registrations, and fee assessments.

Current Version: 1.0.0 (Development Phase)
Last Updated: UI improvements to registration forms and application review workflows

Student Enrollment Workflow

1. Student Registration

Students create an account and submit personal information to initiate the enrollment process.

2. Document Submission

Students upload required documents based on their student type (new, old, transferee, or shifting).

3. Queue Assignment

Students are assigned to a virtual queue for processing by student assistants and registrars.

4. Document Verification

Registrars validate submitted documents and mark them as complete or request additional information.

5. Application Review

Registrars review the complete application and either approve for enrollment or reject with a detailed reason.

6. Course Registration

Students select courses based on their program requirements and available schedules.

6. Fee Assessment

The system calculates tuition and other applicable fees based on enrolled courses and student status.

7. Payment Processing

Students select payment options (full payment or installment) and submit proof of payment.

8. Enrollment Confirmation

Upon successful payment verification, enrollment is confirmed, and students receive their class schedules.

Database Structure

The system uses a relational database with the following core tables:

students

Stores student personal and academic information

  • id (Primary Key)
  • first_name
  • last_name
  • email
  • contact
  • address
  • type (new, old, transferee, shifting)
  • course
  • year_level
  • created_at
  • updated_at
documents

Tracks student document submissions

  • id (Primary Key)
  • student_id (Foreign Key)
  • name
  • type
  • file_path
  • status (Submitted, Pending, Missing)
  • date_submitted
  • verified_by
  • verification_date
enrollments

Records student enrollment details

  • id (Primary Key)
  • student_id (Foreign Key)
  • semester
  • school_year
  • section
  • date_enrolled
  • status
  • approved_by
  • approval_date
courses

Catalog of available courses

  • id (Primary Key)
  • code
  • description
  • units
  • type (Lec, Lab)
  • prerequisites
class_schedules

Course schedules and sections

  • id (Primary Key)
  • course_id (Foreign Key)
  • section
  • days
  • start_time
  • end_time
  • room
  • instructor
  • max_students
fees

Tuition and other fee information

  • id (Primary Key)
  • enrollment_id (Foreign Key)
  • type
  • amount
  • description
  • payment_status
